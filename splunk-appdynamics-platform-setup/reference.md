Virtual Appliance VMware operator coverage:

- vSphere and ESXi OVA pages are currently published under 25.10 paths, while
  the On-Premises platform, Enterprise Console, and Controller command-line
  pages expose 26.4 paths. Do not relabel the VMware OVA docs as 26.4.
- vSphere deployment requires three VM deployments from the OVA/OVF flow and
  host configuration values for host name, host IP/subnet, default gateway,
  DNS IP, and domain name.
- ESXi deployment requires deploying the OVA-backed VMs and then bootstrapping
  the node with `sudo appdctl host init` so the host details can be supplied.
- The rendered VMware scripts are dry-run first. OVF Tool omits passwords from
  the target locator and prompts interactively; govc reads a password file only
  when `VMWARE_APPLY=1`.

Owned surfaces:

- On-Premises Controller lifecycle
- Virtual Appliance lifecycle
- On-Premises overview, deployment planning, platform quickstart, and release/reference review
- Deployment-method selection and frictionless user routing
- Enterprise Console inventory and managed-component changes
- Events Service, EUM Server, and Synthetic Server planning
- HA, backup, restore, and upgrade runbooks
- TLS and secure platform hardening

Enterprise Console mutation is never implicit. Use
`--accept-enterprise-console-mutation` only after reviewing the rendered host
inventory and apply plan.

The Enterprise Console CLI accepts password arguments for login and some
Controller jobs. This skill does not render password values or read password
files into shell arguments. The command plan assumes an existing authenticated
Enterprise Console CLI session or a separately approved local credential wrapper.
